1.0 Scope & Purpose

This Quality Manual outlines the Quality Management System (QMS) for NEXORA Technologies. It governs the design, development, manufacture, and cloud deployment of the MediTwin Bio-Patch (Class IIa Software as a Medical Device).

The scope encompasses:

  • Hardware: ESP32-based microcontroller integration with MAX30102 (SpO2) and AD8232 (ECG) sensors.
  • Firmware: Embedded C++ logic for 500Hz data sampling and I2C communication.
  • Cloud Software: Python (FastAPI) backend, Pydantic validation layers, and LSTM predictive models.

2.0 Normative References

The system is designed in strict accordance with the following international standards:

  • ISO 13485:2016 - Medical devices — Quality management systems.
  • ISO 14971:2019 - Application of risk management to medical devices.
  • IEC 62304:2006 - Medical device software — Software life cycle processes.
  • IEC 60601-1-2 - Medical electrical equipment — Electromagnetic disturbances.

4.0 Software Lifecycle (IEC 62304)

The MediTwin platform is classified as Software Safety Class B (Non-serious injury possible). Development follows the V-Model lifecycle.

4.1 Version Control & CI/CD

All code is managed via Git with strict branch protection rules.

  • Master Branch: Production-ready code (Locked). Requires 2 senior approvals.
  • Dev Branch: Integration testing environment.
  • Firmware Tagging: Semantic versioning (e.g., v2.4.1-esp32) corresponds to hardware revisions.

5.2 Performance Thresholds

A model candidate is only pushed to production if it meets the following KPIs on the "Blind Test" set:

  • Arrhythmia Sensitivity: > 94.0%
  • False Positive Rate: < 2.5%
  • Inference Latency: < 200ms (Cloud), < 50ms (Edge)
